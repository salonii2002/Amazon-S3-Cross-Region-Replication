Project Overview

Objective

To demonstrate how Amazon S3 automatically replicates data from one AWS region to another for backup and disaster recovery.

Problem Statement

Single-region storage may cause data loss during outages. This project ensures data safety using multi-region replication.

Solution

Amazon S3 Cross-Region Replication copies objects automatically without manual intervention.

Architecture

Flow:

User uploads file → Source S3 Bucket (Region A) → Replication Rule → Destination S3 Bucket (Region B)

How It Works

  • Two S3 buckets are created in different AWS regions.
  • Versioning is enabled on both buckets.
  • A replication rule is configured on the source bucket.
  • IAM role allows secure replication between buckets.
  • When a file is uploaded, AWS replicates it automatically.

Security & Safety

  • ✔ Buckets are private with Block Public Access enabled
  • ✔ IAM roles follow least-privilege principle
  • ✔ Server-side encryption protects stored data
  • ✔ Versioning prevents accidental data loss
  • ✔ AWS managed infrastructure ensures compliance
